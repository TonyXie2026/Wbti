# 信息架构与页面结构

## 1. 路由建议

如果用 Next.js：

- `/` 首页
- `/test` 测试页
- `/result` 结果页

如果用单页应用：

- LandingSection
- QuestionSection
- ResultSection

## 2. 首页结构

### 顶部区域

- Logo / 小标题（可选）
- 主标题
- Slogan
- 开始按钮

### 推荐文案层级

- 主标题：测测你是哪种互联网人
- Slogan：互联网自有一套分人方法
- 辅助文案：不一定科学，但很适合认领

## 3. 测试页结构

### 头部

- 进度条
- 当前题号，如 `03 / 12`

### 中间

- 问题文本

### 底部

- 4 个选项按钮
- 上一题按钮（可选）

## 4. 结果页结构

### 结果卡片

- 提示文案：你的互联网身份是
- 标签中文名
- 标签英文代号
- 角色图
- 一句话简介

### 操作区

- 再测一次
- 保存截图（可选）
- 分享给朋友（可选）

## 5. 视觉建议

### 首页

- 大面积留白
- 中央聚焦布局
- Slogan 比标题略小，但要显眼

### 测试页

- 白底卡片式布局
- 每题停留感强
- 选项按钮要大，移动端易点按

### 结果页

- 类似测试卡片
- 形象图居中
- 标签字号最大
- 简介放底部

## 6. UI 风格关键词

- 轻量
- 简洁
- 卡片式
- 少量圆角
- 偏网页测试站
- 不做复杂赛博效果

## 7. 配色建议

推荐做淡色背景 + 单一强调色。

示例：

- 背景：#eef3ef
- 卡片：#ffffff
- 标题：#333333
- 强调色：#6aa84f 或 #4f8f5f

## 8. 组件建议

- `LandingHero`
- `StartButton`
- `QuestionCard`
- `OptionButton`
- `ProgressBar`
- `ResultCard`
- `CharacterImage`
- `RetakeButton`
